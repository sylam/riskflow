# Adaptive corridor — eval sweep + selector verdict

## VERDICT: the fenced in-model u signal has NO causal cross-fence selection skill — FAIL vs the success bar.

Structural finding: fenced u_mean is MONOTONE in fence tightness — u(b040) >= u(b060) >= u(free)
on 46/48 months (the 2 exceptions pick b060; free never wins). Tightening the fence always
improves the frozen value fn's in-sim rollout (it cuts off extrapolation / winner's-curse actions
— a within-model regularization effect), so the u-LEVEL carries no cross-month information about
which fence wins on that month's single realized path. The u-MARGIN doesn't either: rank
correlation of u-margin vs realized margin is ~0 (spearman -0.06 / +0.03 / -0.06 across the three
fence pairs), and selector hit rate vs oracle is 29-31% ~= the 33% chance level.

Consequences: the RAW selector degenerates to (almost) fixed 0.40 (+18.65 / 48.4 — fails the
mean bar); the GUARDED selector (a-priori 1-sigma threshold) merely interpolates between fixed
0.40 and fixed 0.60 (+21.15 / 52.5 — fails the mean bar), and NO threshold in the sensitivity
table produces mean > +23.10: the sensitivity column is monotone toward fixed 0.60. The
Phase B two-month "evidence" (0.40 ranked above 0.60 on 202105/202102) is exactly what uniform
monotonicity produces and was not selection skill. The oracle headroom (+43.56/72.2) is real but
this signal cannot reach it; band selection needs information the trained value fn does not have.

48 months x 3 fences (band 0.40 / 0.60 / free), eval-only ensemble verdicts (frozen corridor-free full48 checkpoints, seeds 7/42/314), batch 2048, common random numbers per month (eval seed 101; fences differ only by Evaluator.Total_Position_Schedule).
